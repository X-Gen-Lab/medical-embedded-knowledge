# 医疗器械嵌入式软件知识体系

## 项目简介

本项目是一个全面的医疗器械嵌入式软件知识管理系统，旨在帮助开发人员、质量工程师和监管人员系统地学习和掌握医疗器械嵌入式软件开发所需的核心知识、技能和认知。

系统采用文档即代码（Docs-as-Code）的方法，使用Markdown格式组织知识内容，通过MkDocs静态站点生成器生成可浏览、可搜索的知识库网站。

## 主要特性

- **全面的知识覆盖**：涵盖核心技术、医疗法规、软件工程实践
- **结构化内容**：清晰的知识模块组织，便于学习和查找
- **定制化学习路径**：为不同角色提供针对性的学习路径
- **实践案例**：包含真实的医疗器械软件开发案例
- **多语言支持**：支持中文和英文两种语言
- **离线访问**：支持PDF导出和离线HTML包
- **版本控制**：所有内容使用Git管理，支持协作和版本追踪

## 快速开始

### 环境要求

- Python 3.8+
- Git

### 安装步骤

1. 克隆仓库：
```bash
git clone <repository-url>
cd Medical
```

2. 安装依赖：
```bash
pip install -r requirements.txt
```

3. 本地预览：
```bash
mkdocs serve
```

4. 在浏览器中访问 `http://127.0.0.1:8000`

### 构建静态站点

```bash
mkdocs build
```

构建后的静态文件将生成在 `site/` 目录中。

## 内容组织

### 核心技术知识
- 嵌入式C/C++编程
- 实时操作系统（RTOS）
- 硬件接口（I2C、SPI、UART等）
- 低功耗设计
- 信号处理与算法

### 医疗法规与标准
- IEC 62304 - 医疗器械软件生命周期过程
- ISO 13485 - 质量管理体系
- ISO 14971 - 风险管理
- FDA法规
- IEC 60601-1 - 医疗电气设备安全
- IEC 81001-5-1 - 网络安全

### 软件工程实践
- 需求工程
- 软件架构设计
- 编码规范（MISRA C、CERT C）
- 测试策略
- 配置管理
- 静态代码分析

## 学习路径

我们为不同角色提供了定制化的学习路径：

- **嵌入式软件工程师**：侧重编程、RTOS和硬件接口
- **质量保证工程师**：侧重法规标准、测试和文档
- **系统架构师**：侧重架构设计、风险管理和系统集成
- **监管事务专员**：侧重法规要求、认证流程和技术文档
